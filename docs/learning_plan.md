# C# 与 Unity 学习计划

目标：服务于“随机地牢刷食材、料理成长、基地升级”的 Unity 学习项目。重点不是成为 C# 专家，而是能看懂 AI 生成的代码、能小改功能、能定位报错。

## C# 学习计划

| 阶段 | 学习内容 | 对应项目用途 |
|---|---|---|
| 第 1 阶段 | 变量、类型、if、for、函数 | 血量、攻击力、食材数量、材料是否足够、制作料理 |
| 第 2 阶段 | 数组、List、Dictionary | 背包、食材库存、配方需求、怪物列表 |
| 第 3 阶段 | class、对象、public/private | 玩家、怪物、料理、食材、基地建筑的数据结构 |
| 第 4 阶段 | 继承、接口基础 | 不同怪物、不同可交互物体、不同料理效果 |
| 第 5 阶段 | 事件、JSON 基础 | 怪物死亡通知掉落、保存背包/属性/基地等级 |
| 第 6 阶段 | 代码阅读训练 | 看懂 AI 生成的 Unity 脚本，能小改和定位报错 |

## Unity 学习计划

| 阶段 | 学习内容 | 对应项目用途 |
|---|---|---|
| 第 1 阶段 | 编辑器、Scene、Hierarchy、Inspector、Project | 看懂 Unity 项目结构 |
| 第 2 阶段 | GameObject、Component、脚本挂载 | 玩家、怪物、食材、基地建筑都是物体加组件 |
| 第 3 阶段 | MonoBehaviour、Start、Update、Transform | 玩家移动、初始化数据、每帧检测输入 |
| 第 4 阶段 | Input、Rigidbody、Collider | 移动、攻击、碰撞、拾取食材 |
| 第 5 阶段 | Prefab、Instantiate、Destroy、Random | 生成怪物、掉落食材、随机地牢内容 |
| 第 6 阶段 | UI：Canvas、Text、Button、Panel | 背包界面、料理按钮、基地升级界面 |
| 第 7 阶段 | Animator、Scene 管理 | 攻击动画、死亡动画、基地/地牢场景切换 |
| 第 8 阶段 | ScriptableObject、存档 | 料理表、怪物表、食材表、保存进度 |
| 第 9 阶段 | Tilemap、随机房间 | 制作 2D 地牢地图和随机地牢结构 |
| 第 10 阶段 | 项目整合 | 跑通完整循环：进地牢、打怪、掉食材、做料理、变强 |

## 推荐节奏

| 周期 | 学习重点 | 项目成果 |
|---|---|---|
| 第 1-2 周 | C# 基础 + Unity 编辑器基础 | 看懂项目结构，做出玩家移动 |
| 第 3-4 周 | 玩家移动、怪物、战斗、掉落 | 玩家能打怪，怪物能掉落食材 |
| 第 5-6 周 | 背包、料理、属性成长、UI | 能拾取食材并制作料理 |
| 第 7-8 周 | 基地升级、场景切换、存档 | 基地和地牢之间形成循环 |
| 第 9-10 周 | 随机地牢、难度层级、整体整合 | 完成一个可玩的核心原型 |

## 第一个月目标

```text
玩家能移动
能打死 1 种怪物
怪物掉落 1 种食材
玩家能拾取食材
回到基地点击按钮制作 1 道料理
吃料理后最大生命值 +10
```
